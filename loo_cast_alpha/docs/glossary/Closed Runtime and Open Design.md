Closed Runtime and Open Design means runtime activation is deterministic and bounded once locked, while architecture
and contract design remain openly evolvable between lock cycles.
This keeps execution stable without pretending design exploration is finished.
Design changes become runtime changes through explicit lifecycle transitions, not hidden hot mutation.
